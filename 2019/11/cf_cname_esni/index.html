Cloudflare CNAME 接入启用 ESNI

Posted on 2019-11-28 Edited on 2022-01-28

由于 ESNI 需要配置 DNS 记录，因此默认情况下 Cloudflare 如果使用 CNAME 接入则没有 ESNI 保护。

目前 Cloudflare 所有网站使用的 ESNI Public Key 是同一个，可以通过任意dig一个 Cloudflare 域名的_esniTXT 记录得到 (如dig -t TXT _esni.cloudflare.com)。

假设要配置 ESNI 的域名为www.example.com，直接添加 CNAME 记录，域名_esni.www.example.com，目标指向_esni.cloudflare.com，即可启用 ESNI。
