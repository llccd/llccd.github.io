SQL Server更改计算机名后设置

Posted on 2019-03-27 Edited on 2020-07-09

更改计算机名后，SQL Server仍能通过Windows身份验证登录，但在执行部分操作时会报用户不存在/计算机名不正确等问题，需要手动进行设置。

  1. 若SQL Server设置为仅Windows身份验证登录，则先启用SQL Server身份验证登录，再启用sa账户并设置密码，重启SQL Server
  2. 使用sa登录,将smss“安全性/登录名”中各个账户用重命名修改为新的计算机名
